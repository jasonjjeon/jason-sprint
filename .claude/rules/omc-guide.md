# OMC 모드 & 에이전트 가이드 규칙

## 현재 OMC 세팅
- 기본 모드: ultrawork (최대 병렬 처리)
- 팀: 3 agents, executor
- Agent Teams: 활성화

## 자동 제안 규칙

사용자의 요청 패턴을 보고, 더 효율적인 OMC 모드가 있으면 제안하세요.
강제하지 말고 "이렇게 실행하면 더 좋을 것 같은데, 해볼까요?" 형태로.

### 제안 타이밍

- 작업이 **5분 이상 걸릴 것 같으면** → autopilot 또는 ralph 제안
- 작업이 **3개 이상 독립적이면** → ultrawork 또는 team 제안
- 작업이 **탐색/조사 성격이면** → researcher 에이전트 제안
- 작업이 **데이터 조회 성격이면** → data-analyst 에이전트 제안
- 작업이 **새 기능/프로젝트 기획·설계 성격이면** → deep-interview 먼저 제안 (요구사항 확정 후 autopilot 전환)

### 제안하지 않을 때

- 간단한 질문 (1-2줄로 답변 가능)
- 이미 특정 모드로 실행 중일 때
- 사용자가 "그냥 해줘"라고 할 때 → 바로 실행

## 상태 표시 규칙 (필수)

VSCode 확장 환경에서는 HUD가 안 보이므로, 텍스트로 상태를 직접 표시합니다.

### 모드/에이전트 전환 시
전환할 때마다 대화 맨 위에 한 줄 상태를 표시하세요:
```
> [OMC] 모드: autopilot | 에이전트: data-analyst | MCP: redash
```

### 장시간 작업 시
진행 상황을 주기적으로 표시하세요:
```
> [OMC] autopilot 3/7 단계 진행 중 | 현재: 스키마 조회
```

### 작업 완료 시
```
> [OMC] 완료 | 소요: 약 2분 | 생성 파일: projects/리서치/경쟁사분석.md
```

### 에이전트 위임 시
```
> [OMC] researcher 에이전트에게 위임합니다 → "무신사 최근 프로모션 조사"
```
